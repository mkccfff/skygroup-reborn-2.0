## 🌐 Деплой (GitHub Pages)
- **Живой сайт: https://mkccfff.github.io/skygroup/** (аккаунт `mkccfff`, репо `mkccfff/skygroup`, public).
- Pages: ветка `main`, root. Сайт = содержимое `dist/` (там свой git-репозиторий + `.nojekyll`).
- ВАЖНО для под-пути `/skygroup/`: пути к ассетам в коде сделаны **относительными** (`assets/...`, не `/assets/...`) — работает и в деве (рут), и на Pages. Сборка только с `--base=./`.

- ⚠️ Токен деплоя был разовый — после правок нужен новый PAT (scope `repo`).
- Видео оставлены в полном качестве (hero 46МБ + about 51МБ = ~97МБ). Галерея ужата до 1920px (50→3.5МБ). Бэкапы медиа (старые видео/фото, оригиналы галереи) — в `_media-backups/` (вне сборки).

## Как запустить (локально)
- Превью: launch.json конфиг `skygroup-reborn-2.0`, порт **5275**. Открывать **http://localhost:5275** (не по сетевому IP — он меняется с Wi-Fi).
- `node_modules` — junction на `skygroup-future\node_modules` (установка не нужна).
- ВАЖНО: если правки «не видны» — на порту завис старый vite. Убить:
  `Get-NetTCPConnection -LocalPort 5275 -State Listen | %{ Stop-Process -Id $_.OwningProcess -Force }` → потом запустить превью заново.
- Перед скриншотом ставить видео на паузу: `document.querySelectorAll('video').forEach(v=>v.pause())`.

## Деплой (файловый сервер + домен) — кратко
1. Сборка: в `skygroup-reborn-2.0` выполнить `npm run build` → папка **`dist/`** (статика: index.html + assets, видео/картинки из public копируются автоматически).
2. Залить **содержимое `dist/`** на сервер в корень сайта (FTP/SFTP/панель), напр. `/var/www/skygroup` или `public_html`.
3. Домен: в DNS регистратора A-запись `@` → IP сервера (и `www` → тот же IP/CNAME).
4. Веб-сервер (nginx): отдавать `dist`, для SPA — `try_files $uri /index.html;`. HTTPS — Let's Encrypt (certbot).
5. Проверить, что большие mp4 отдаются (лимиты сервера), по возможности дожать видео.
